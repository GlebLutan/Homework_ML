"""Классификатор мягкого голосования/правила большинства для неподогнанных оценивателей.

    Подробнее читайте в :ref:`Руководстве пользователя <voting_classifier>`.

    .. versionadded:: 0.17

    Параметры
    ----------
    estimators : список кортежей (str, estimator)
        Вызов метода ``fit`` для ``VotingClassifier`` будет подгонять копии
        исходных оценивателей, которые будут сохранены в атрибут класса
        ``self.estimators_``. Оценивателя можно установить в ``'drop'`` 
        используя :meth:`set_params`.

        .. versionchanged:: 0.21
            ``'drop'`` принимается. Использование None было прекращено 
            в версии 0.22 и удалено в версии 0.24.

    voting : {'hard', 'soft'}, по умолчанию='hard'
        Если 'hard', использует предсказанные метки классов для голосования 
        по правилу большинства. Если 'soft', предсказывает метку класса на 
        основе argmax суммы предсказанных вероятностей, что рекомендуется 
        для ансамбля хорошо откалиброванных классификаторов.

    weights : массиво-подобный, форма (n_classifiers,), по умолчанию=None
        Последовательность весов (`float` или `int`) для взвешивания 
        предсказанных меток классов (голосование 'hard') или вероятностей 
        класса перед усреднением (голосование 'soft'). Использует 
        одинаковые веса при `None`.

    n_jobs : int, по умолчанию=None
        Количество заданий для параллельного выполнения ``fit``.
        ``None`` означает 1, если только не используется контекст 
        :obj:`joblib.parallel_backend`. ``-1`` означает использование 
        всех процессоров. Подробнее см. :term:`Глоссарий <n_jobs>`.

        .. versionadded:: 0.18

    flatten_transform : bool, по умолчанию=True
        Влияет на форму выходных данных преобразования только при voting='soft'.
        Если voting='soft' и flatten_transform=True, метод transform возвращает
        матрицу формы (n_samples, n_classifiers * n_classes). При
        flatten_transform=False возвращает
        (n_classifiers, n_samples, n_classes).

    verbose : bool, по умолчанию=False
        Если True, прошедшее время во время подогонки будет выведено по 
        завершении.

        .. versionadded:: 0.23

    Атрибуты
    ----------
    estimators_ : список классификаторов
        Коллекция подогнанных под-оценивателей, как определено в 
        ``estimators``, которые не являются 'drop'.

    named_estimators_ : :class:`~sklearn.utils.Bunch`
        Атрибут для доступа к любым подогнанным под-оценивателям по имени.

        .. versionadded:: 0.20

    le_ : :class:`~sklearn.preprocessing.LabelEncoder`
        Трансформер, используемый для кодирования меток во время подогонки 
        и декодирования во время предсказания.

    classes_ : ndarray, форма (n_classes,)
        Метки классов.

    n_features_in_ : int
        Количество признаков, виденых во время :term:`fit`. Определено 
        только если базовый классификатор предоставляет такой атрибут 
        при подогонке.

        .. versionadded:: 0.24

    feature_names_in_ : ndarray, форма (`n_features_in_`,)
        Имена признаков, виденых во время :term:`fit`. Определено только если
        базовые оценивватели предоставляют такой атрибут при подогонке.

        .. versionadded:: 1.0

    Смотрите также
    --------
    VotingRegressor : Регрессор с голосованием для предсказания.
"""
