Дерево решений для регрессии.

Подробнее см. в :ref:`Руководстве пользователя <tree>`.

Параметры
----------
criterion : {"squared_error", "friedman_mse", "absolute_error", \
        "poisson"}, default="squared_error"
    Функция для оценки качества разбиения. Поддерживаемые критерии:
    "squared_error" — среднеквадратичная ошибка, эквивалентная
    уменьшению дисперсии как критерию выбора признака и минимизирует L2
    потерю, используя среднее значение в каждом терминальном узле;
    "friedman_mse" — использует среднеквадратичную ошибку с показателем
    улучшения Фридмана для потенциальных разбиений;
    "absolute_error" — средняя абсолютная ошибка, минимизирующая L1-потерю,
    используя медиану в каждом терминальном узле; и "poisson" — использует
    снижение полусредней девиации Пуассона для поиска разбиений.

    .. versionadded:: 0.18
       Критерий средней абсолютной ошибки (MAE).

    .. versionadded:: 0.24
        Критерий девиации Пуассона.

splitter : {"best", "random"}, default="best"
    Стратегия выбора разбиения в каждом узле. Поддерживаемые стратегии:
    "best" — выбрать наилучшее разбиение, "random" — выбрать лучшее
    случайное разбиение.

max_depth : int, default=None
    Максимальная глубина дерева. Если None, узлы расширяются до тех пор,
    пока все листья не станут чистыми или пока в листьях не окажется меньше
    образцов, чем `min_samples_split`.

    Пример влияния ``max_depth`` на модель см. в
    :ref:`sphx_glr_auto_examples_tree_plot_tree_regression.py`.

min_samples_split : int or float, default=2
    Минимальное число образцов, необходимое для разбиения внутреннего узла:

    - Если int, считать `min_samples_split` минимальным числом.
    - Если float, `min_samples_split` — доля, и
      `ceil(min_samples_split * n_samples)` — минимальное число образцов для
      каждого разбиения.

    .. versionchanged:: 0.18
       Добавлена поддержка дробных значений.

min_samples_leaf : int or float, default=1
    Минимальное число образцов в листовом узле. Разбиение на любой глубине
    будет рассмотрено только если в левом и правом потомках останется как
    минимум ``min_samples_leaf`` обучающих образцов. Это может сглаживать
    модель, особенно в регрессии.

    - Если int, считать `min_samples_leaf` минимальным числом.
    - Если float, `min_samples_leaf` — доля, и
      `ceil(min_samples_leaf * n_samples)` — минимальное число образцов для
      каждого узла.

    .. versionchanged:: 0.18
       Добавлена поддержка дробных значений.

min_weight_fraction_leaf : float, default=0.0
    Минимальная взвешенная доля от суммы весов всех входных образцов,
    необходимая для листового узла. Образцы имеют равный вес, если
    `sample_weight` не задан.

max_features : int, float or {"sqrt", "log2"}, default=None
    Число признаков, рассматриваемых при поиске лучшего разбиения:

    - Если int, рассматриваются `max_features` признаков при каждом
      разбиении.
    - Если float, `max_features` — доля, и
      `max(1, int(max_features * n_features_in_))` признаков учитывается при
      каждом разбиении.
    - Если "sqrt", то `max_features=sqrt(n_features)`.
    - Если "log2", то `max_features=log2(n_features)`.
    - Если None, то `max_features=n_features`.

    Примечание: поиск разбиения не останавливается, пока не будет найдена
    хотя бы одна допустимая разбивка, даже если для этого потребуется
    фактически просмотреть больше, чем ``max_features`` признаков.

random_state : int, RandomState instance or None, default=None
    Управляет случайностью оценщика. Признаки всегда случайно перемешиваются
    при каждом разбиении, даже если ``splitter`` установлен в ``"best"``.
    Когда ``max_features < n_features``, алгоритм будет случайно выбирать
    ``max_features`` признаков при каждом разбиении перед поиском лучшего
    среди них. Однако лучшее найденное разбиение может отличаться между
    запусками, даже если ``max_features=n_features``. Это происходит, если
    улучшение критерия одинаково для нескольких разбиений и одно из них
    выбирается случайно. Для детерминированного поведения при обучении
    `random_state` должен быть фиксирован целым числом. Смотрите
    :term:`Glossary <random_state>` для подробностей.

max_leaf_nodes : int, default=None
    Расти дерево с ограничением по числу листьев ``max_leaf_nodes`` в
    порядке "best-first". Лучшие узлы определяются относительным снижением
    нечистоты. Если None, число листьев не ограничено.

min_impurity_decrease : float, default=0.0
    Узел будет разделён, если это разбиение приводит к снижению нечистоты,
    большему или равному этому значению.

    Формула взвешенного снижения нечистоты::

        N_t / N * (impurity - N_t_R / N_t * right_impurity
                            - N_t_L / N_t * left_impurity)

    где ``N`` — общее число образцов, ``N_t`` — число образцов в текущем
    узле, ``N_t_L`` — число образцов в левом потомке, а ``N_t_R`` — в правом
    потомке.

    ``N``, ``N_t``, ``N_t_R`` и ``N_t_L`` относятся к взвешенным суммам, если
    передан ``sample_weight``.

    .. versionadded:: 0.19

ccp_alpha : non-negative float, default=0.0
    Параметр сложности, используемый для минимальной стоимости-сложности при
    обрезке (Minimal Cost-Complexity Pruning). Поддерево с наибольшей
    сложностью стоимости, меньшей чем ``ccp_alpha``, будет выбрано. По
    умолчанию обрезка не выполняется. См. :ref:`minimal_cost_complexity_pruning`
    для подробностей. Пример показан в
    :ref:`sphx_glr_auto_examples_tree_plot_cost_complexity_pruning.py`.

    .. versionadded:: 0.22

monotonic_cst : array-like of int of shape (n_features), default=None
    Указывает ограничения монотонности для каждого признака:
      - 1: монотонное возрастание
      - 0: без ограничения
      - -1: монотонное убывание

    Если `monotonic_cst` равно None, ограничения не применяются.

    Ограничения монотонности не поддерживаются для:
      - многовыходной регрессии (т.е. когда `n_outputs_ > 1`),
      - регрессии, обученной на данных с пропущенными значениями.

    Подробнее см. в :ref:`User Guide <monotonic_cst_gbdt>`.

    .. versionadded:: 1.4

Атрибуты
----------
feature_importances_ : ndarray of shape (n_features,)
    Значимость признаков. Чем выше, тем более важен признак. Значимость
    вычисляется как (нормализованное) суммарное снижение критерия, вносимое
    данным признаком (так называемая Gini importance).

    Внимание: основанные на нечистоте оценки значимости признаков могут быть
    вводящими в заблуждение для признаков с высокой кардинальностью. В
    качестве альтернативы смотрите :func:`sklearn.inspection.permutation_importance`.

max_features_ : int
    Вычисленное значение `max_features`.

n_features_in_ : int
    Число признаков, встреченных при :term:`fit`.

    .. versionadded:: 0.24

feature_names_in_ : ndarray of shape (`n_features_in_`,)
    Имена признаков, встреченные при :term:`fit`. Определено только если у
    `X` есть имена признаков и все они строки.

    .. versionadded:: 1.0

n_outputs_ : int
    Число выходов при выполнении ``fit``.

tree_ : Tree instance
    Внутренний объект `Tree`. Смотрите ``help(sklearn.tree._tree.Tree)`` для
    атрибутов объекта Tree и :ref:`sphx_glr_auto_examples_tree_plot_unveil_tree_structure.py`
    для примеров использования.

Notes
-----
Значения параметров по умолчанию, управляющих размером деревьев (например,
``max_depth``, ``min_samples_leaf`` и т.д.) приводят к полностью
выращиваемым и необрезанным деревьям, которые могут быть очень большими на
некоторых наборах данных. Для уменьшения потребления памяти рекомендуется
контролировать сложность и размер деревьев, задавая соответствующие
значения параметров.

Examples
--------
>>> from sklearn.datasets import load_diabetes
>>> from sklearn.model_selection import cross_val_score
>>> from sklearn.tree import DecisionTreeRegressor
>>> X, y = load_diabetes(return_X_y=True)
>>> regressor = DecisionTreeRegressor(random_state=0)
>>> cross_val_score(regressor, X, y, cv=10)
...                    # doctest: +SKIP
...

       0.16...,  0.11..., -0.73..., -0.30..., -0.00...])
